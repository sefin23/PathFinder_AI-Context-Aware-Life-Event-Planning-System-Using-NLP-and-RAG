The problem right now

User sees "Download e-Aadhaar." They open Google, search for UIDAI, click wrong link, get confused, abandon. 15 minutes wasted on a 2-minute task. PathFinder knows the exact URL. It's sitting on data it's not using.

What PathFinder should do

Open the right page. Show what to click. Pre-fill what it already knows (your Aadhaar number from your profile, your name from your vault). Tell you exactly what to download. That's the difference between a list and a navigator.

The one-liner: "Google tells you that you need an Aadhaar update. PathFinder opens the right page, shows you what to click, fills in what it knows, and tells you exactly what to download when you're done."

Every task gets two modes

🗺 Guide me →

PathFinder walks the user through completing this task right now. Step-by-step, inside the app. Shows the exact URL, what to click, pre-fills known details, tells them what to download. User never has to search. For users who don't know where to start.

✓ Mark done

For tasks the user already completed, or prefers to handle themselves. One tap, task checked off. No friction, no required flow. For experienced users or tasks they handle regularly.

V1 — what's realistic without APIs

Every task gets a structured guide built once in the database. No government API access needed, no browser automation. Just well-researched, accurate guidance. This alone cuts cognitive load by ~60%.

1. Exact URL
Not "visit UIDAI website." The specific deep-link to the exact page. Researched once, maintained in a task_guides DB table.
2. 3-step visual guide
Screenshots of exactly what to click. Stored as annotated images. User sees it before they even open the page.
3. Pre-filled details
Name, DOB, address from vault documents. Shown alongside the form so user just copies — doesn't have to remember.
4. Expected result
"You'll receive an SMS within 2 hours. The PDF will have a QR code in the top right." User knows what success looks like.
5. What to save
"Save this PDF. Rename it 'Aadhaar_2026.pdf'. You'll need it in Step 7 of this plan." Eliminates the "where did I save that" problem.
V2+ (harder)
In-app browser with overlays, form auto-fill via browser extension, actual form submission. Needs official API or RPA. Phase 3.

How to build it

Database: task_guides table

One row per task type. Content is curated once by the team, not AI-generated. The AI just picks the right guide for each task in a plan.

SQLtask_guides.sql
CREATE TABLE task_guides (
  id              UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  task_type       VARCHAR(100) NOT NULL UNIQUE,  -- e.g. "download_aadhaar"
  title           VARCHAR(255) NOT NULL,
  estimated_mins  INTEGER,                         -- "~5 minutes"
  url             TEXT,                             -- exact deep-link
  steps           JSONB        NOT NULL,           -- array of step objects
  prefill_fields  JSONB,                           -- which user fields to show
  expected_result TEXT,                             -- "you'll receive an SMS..."
  what_to_save    TEXT,                             -- "save as PDF, rename..."
  tags            TEXT[],                           -- ["identity", "aadhaar"]
  updated_at      TIMESTAMP    DEFAULT now()
);

-- steps JSONB structure:
-- [{
--   "num": 1,
--   "title": "Go to the UIDAI Aadhaar download page",
--   "description": "This is the official government page...",
--   "action": { "type": "open_url", "url": "https://myaadhaar.uidai.gov.in/", "label": "Open UIDAI page" },
--   "screenshot_url": "https://assets.pathfinder.app/guides/aadhaar_step1.png"
-- }]

-- prefill_fields JSONB structure:
-- [{ "label": "Aadhaar number", "source": "profile.aadhaar_number" },
--  { "label": "Full name",      "source": "profile.full_name" }]
Pythonapi/guides.py
@router.get('/tasks/{task_id}/guide')
async def get_task_guide(task_id: str, user=Depends(get_current_user)):
    """
    Returns the guide for a specific task, with prefilled values
    from the user's vault and profile.
    """
    task = db.get_task(task_id)
    guide = db.get_guide_by_type(task.task_type)

    if not guide:
        return {'has_guide': False}

    # Resolve prefill values from user profile + vault
    prefilled = []
    for field in (guide.prefill_fields or []):
        value = resolve_field(field['source'], user)
        prefilled.append({
            'label': field['label'],
            'value': value,
            'source_label': get_source_label(field['source'])
        })

    return {
        'has_guide': True,
        'title': guide.title,
        'estimated_mins': guide.estimated_mins,
        'url': guide.url,
        'steps': guide.steps,
        'prefilled': prefilled,
        'expected_result': guide.expected_result,
        'what_to_save': guide.what_to_save
    }


def resolve_field(source: str, user) -> str:
    """Resolve a dotted field path from user profile or vault."""
    parts = source.split('.')
    if parts[0] == 'profile':
        return getattr(user.profile, parts[1], None)
    elif parts[0] == 'vault':
        doc = user.get_vault_doc_by_type(parts[1])
        return doc.extracted_fields.get(parts[2]) if doc else None
    return None

Vault document reading

When a user uploads a document, extract structured data from it using a PDF extraction + LLM pipeline. Store the extracted fields so they can be used everywhere.

Pythonvault/reader.py — document intelligence
async def process_vault_upload(doc_id: str, file_path: str, doc_type: str):
    """
    Called after every vault upload. Extract structured data,
    update user profile, trigger plan updates.
    """
    # 1. Extract text from PDF
    text = extract_pdf_text(file_path)

    # 2. Ask Claude to extract structured fields
    PROMPTS = {
        'offer_letter': """Extract from this offer letter:
            - joining_date (ISO format)
            - company_name
            - office_city
            - role_title
            - employer_specific_requirements (list any docs mentioned)
            Return as JSON.""",
        'aadhaar': """Extract from this Aadhaar:
            - full_name
            - dob (ISO format)
            - aadhaar_number (last 4 digits only for security)
            - address_city
            Return as JSON.""",
    }

    prompt = PROMPTS.get(doc_type)
    if not prompt:
        return

    extracted = await call_claude(prompt + "\n\n" + text[:3000])

    # 3. Store extracted fields on the vault document
    db.update_vault_doc(doc_id, {'extracted_fields': extracted})

    # 4. Apply to user profile if relevant
    if doc_type == 'aadhaar':
        db.update_profile(user_id, {
            'full_name': extracted.get('full_name'),
            'dob': extracted.get('dob'),
        })

    # 5. Trigger plan updates if offer letter
    if doc_type == 'offer_letter':
        await update_plan_from_offer_letter(user_id, extracted)

    # 6. Run mismatch checks
    await run_mismatch_checks(user_id, doc_type, extracted)
